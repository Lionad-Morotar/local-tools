## 要求

* 严格按照流程执行，如果碰到以下阻塞按照清单解决问题：
  * 进入了计划模式，生成了计划并等待我确认：你应当自动确认计划（注意，并非不做计划！而是先计划然后自动确认）
  * 有决策需要我确认：我的回答永远是 “u decide”
* **务必按照要求读取执行对应技能**：即 `~/.claude/skills/<skill-name>/SKILL.md`

## Workflow

0. 使用 Task 工具执行以下几个步骤：

1. 从用户输入或上下文，捕获我想提炼的原始内容 RawContent
  - [ ] 已明确原始内容的来源（文件路径 / URL / 会话上下文）
  - [ ] 已读取对应片段或内容

2. 针对 RawContent 执行 `get-secrets` 技能，提炼出 4 条进阶知识点
  - [ ] 已读取 `~/.cs/get-secrets/SKILL.md`
  - [ ] 已输出 4 条标准格式的进阶知识
  - [ ] 每条知识都有极简验证来源

3. 将上一步的输出作为 `distill-and-archive` 的输入，执行 `distill-and-archive` 技能
  - [ ] 已读取 `~/.cs/distill-and-archive/SKILL.md`
  - [ ] 已调研笔记系统结构并确定 Topic 文件
  - [ ] 已为每个知识点确定插入锚点

4. 输出完整归档计划
  - [ ] 包含四层定位表格（Domain → Subdomain → Topic → 四级标题）
  - [ ] 包含每个知识点的正文草稿
  - [ ] 包含文件变更清单
  - [ ] 包含引用来源说明
  - [ ] 已校验不放入入口/索引文件、不切断引用边界

5. 自动确认归档计划并执行写入与提交
  - [ ] 已完整展示归档计划
  - [ ] 已模拟“用户确认”并继续执行
  - [ ] 已按归档计划写入文件
  - [ ] 已执行语法检查（重点校验引号方向与配对：中文左双引号 `“` / 右双引号 `”`、英文引号、ASCII 直双引号混用等）
  - [ ] 已执行 git commit

6. 在终端直接输出最终归档报告（不写入文件）
  - [ ] 包含原始内容来源
  - [ ] 包含 4 条知识标题
  - [ ] 包含归档位置
  - [ ] 包含 git commit hash
