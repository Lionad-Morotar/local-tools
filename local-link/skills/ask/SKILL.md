---

用户询问了一些问题（或简单列出了关键词，默认为“简单解释”），可能提供了文档路径：意味着他在阅读某个文档时碰到的问题。

# Workflow

1. 根据需要，简单搜索项目代码或互联网
2. 重新对问题进行排列，以便组织回答
3. 回答用户，输出 markdown 到终端：
  3.1 不应反问用户
  3.2 回答要结构化但不应当有复杂嵌套（嵌套不超过 2 级）
  3.3 回答内容要克制对着重号的使用
  3.4 如果需要，在回答末尾询问用户是否更新文档
    3.4.1 仅更新错误性内容或进行补全，**切记不要动主要结构（比如新增错误澄清的章或节）**，除非用户明确要求你这样处理
