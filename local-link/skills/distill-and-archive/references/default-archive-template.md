## 默认归档模版

```markdown
#### [知识点标题，简洁、具体、无元描述标签]

[内容：用自然段落叙述。不出现"L1 层面"、"L2 层面"、"L3 层面"、"作者主张"、"有分析认为"等验证标签。事实性内容直接陈述；机制性内容用"其机制在于"、"这反映了"、"根源在于"等过渡；建议性内容用"工程上应"、"正确做法是"、"避免"等表达。每行不超过 120 字。]

[极少数知识点需要第二段补充竞争性框架、关键语境，或纠正常见误解。]

见：[链接标题](url)
```

## 错误示例 vs 正确示例

### 错误示例（L 标签污染正文）

```markdown
#### temperature 并非越高越好

L1 层面，过高的 temperature 下模型可能不是因为走了另一条合理解题路径而给出不同答案，而是单纯在胡言乱语。L2 层面，采样多样性与推理有效性之间存在非线性关系。L3 层面的工程正确做法是：不同任务的甜蜜点不同。
```

### 正确示例（L 层级内化到语气）

```markdown
#### temperature 并非越高越好

Self-Consistency 依赖多次采样的多样性来覆盖不同推理路径，但 temperature 过高会引入与问题无关的噪声，反而降低投票有效性。过高的 temperature 下，模型可能不是因为走了另一条合理解题路径而给出不同答案，而是单纯生成胡言乱语。这些噪声路径增加了答案分布的离散度，却不贡献有效纠错。不同任务的甜蜜点不同，数学题可能在 0.5–0.7，开放式任务可能略高，但需要通过实验确定，而不是盲目拉到 1.0 以上。
```
