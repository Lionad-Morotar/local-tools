## 插入锚点与边界指南

本指南约束 `distill-and-archive` 在执行归档时如何定位新知识点，避免切断现有段落的结构边界。

## 核心规则

1. **每个新知识点必须给出插入锚点**
   - 前序四级标题：`#### 前序知识点标题`
   - 后序四级标题：`#### 后序知识点标题`
   - 插入位置：后序标题之前，前序标题内容结束之后。

2. **不要把新知识点插入到现有段落与其引用脚注之间**
   - 每个 `####` 知识点只应有一个 `见：[链接](地址)` 来源行，放在该知识点正文结束之后。
   - 若前序知识点以 `见：` 或 `[^n]: ...` 结尾，新知识点应放在这些引用/脚注之后。
   - 错误示例：
     ```markdown
     ...监控实际保留的候选词数量（effective vocabulary size）而非只看超参数值。

     #### temperature 并非越高越好

     ...

     见：[Prompt Engineering Guide](https://www.promptingguide.ai/zh)
     ```
   - 正确示例：
     ```markdown
     ...监控实际保留的候选词数量（effective vocabulary size）而非只看超参数值。

     见：[Prompt Engineering Guide](https://www.promptingguide.ai/zh)

     #### temperature 并非越高越好

     ...
     ```

3. **不要把新知识点插入到脚注定义之间**
   - 若目标区域包含 `[^1]:`、`[^2]:` 等 Markdown 脚注定义，新知识点应放到整块脚注定义之后。

4. **主代理执行前必须校验锚点**
   - 读取目标文件片段，确认 subagent 给出的前序/后序标题真实存在。
   - 确认插入位置不会切断引用、脚注、列表或代码块。
   - 若锚点缺失或存在边界风险，退回 subagent 修正。

## 快速检查清单

- [ ] 前序四级标题在目标文件中存在
- [ ] 后序四级标题在目标文件中存在
- [ ] 新知识点不会插入到 `见：` 引用行与其所属段落之间
- [ ] 新知识点不会插入到 `[^n]:` 脚注定义之间
- [ ] 新知识点不会插入到未闭合的代码块或列表中间
